# Mindustry 世界观与美术设计文档

## 文档概述

本文档深入阐述Mindustry的世界观设定、星球体系、背景故事以及美术设计理念，为游戏的视觉呈现和叙事框架提供完整的设计指导。

## 1. 世界观设计理念

### 1.1 核心设计哲学

Mindustry的世界观构建遵循**科学幻想与工业美学**的融合理念，创造了一个既有科技感又保持可信度的未来工业世界。

#### 设计原则
1. **硬科幻基础**：基于现实科学原理的技术延伸
2. **工业美学**：强调功能性与效率的视觉表达
3. **极简主义**：用最简洁的元素传达最丰富的信息
4. **一致性维护**：所有视觉元素遵循统一的设计语言

#### 世界观核心概念
- **资源驱动的文明**：所有冲突都围绕资源控制展开
- **技术差异化发展**：不同星球发展出独特的技术路线
- **机械与有机的对立**：代表不同的生存哲学
- **探索与征服的循环**：推动故事发展的核心动力

### 1.2 技术架构支持

**核心类**：`Planet.java`、`SectorPreset.java`、`PlanetGenerator.java`

```java
// 星球定义系统

    public Seq<Sector> sectors;      // 区域列表
```

## 2. 宇宙设定与星球体系

### 2.1 宇宙架构设计

#### 太阳系设定
**技术实现**：`Planets.java`

Mindustry的宇宙围绕一个**多星球太阳系**构建，每个星球都有独特的环境特征和文明发展路径。

**星球层级结构**：
```
太阳 (Sun)
├── Serpulo - 原始母星，传统工业文明
├── Erekir - 晶体星球，能量驱动文明
├── Tantros - 冰冻星球，极地生存挑战
├── Gier - 沙漠星球，资源稀缺环境
├── Notva - 海洋星球，流体工程主导
└── Verilus - 森林星球，生物技术发达
```

#### 星球间关系设定
1. **距离设定**：影响运输成本和探索难度
2. **资源互补**：不同星球的资源分布形成贸易需求
3. **技术传播**：先进技术在星球间的扩散路径
4. **冲突根源**：资源竞争导致的星际冲突

### 2.2 Serpulo - 工业母星

#### 星球特征
**环境设定**：
- **地貌类型**：大陆为主，适度的海洋分布
- **气候条件**：温带气候，适宜工业发展
- **资源分布**：均衡的基础资源，适合初期发展
- **生态系统**：工业化程度高，自然环境已大量改造

**技术实现**：`SerpuloPlanetGenerator.java`

#### 文明特点
1. **工业传统**：历史悠久的工业制造文明
2. **资源导向**：以资源开采和加工为核心的经济体系
3. **机械美学**：崇尚机械的功能性和可靠性
4. **保守稳健**：技术发展相对稳定，注重实用性

#### 视觉风格定义
- **色彩基调**：金属灰、工业蓝、铜锈色
- **材质特征**：金属、混凝土、工业涂料
- **建筑风格**：功能主义，简洁实用
- **环境氛围**：工业感浓重，机械运转的律动感

#### 代表性内容
**资源体系**：
- 基础资源：铜、铅、沙子、煤炭
- 工业材料：石墨、硅、玻璃、钛合金
- 高级合金：相涌合金、相织物、爆破混合物

**建筑特色**：
- 传统传送带系统
- 钻头和工厂生产线
- 常规能源发电机
- 基础防御炮塔

### 2.3 Erekir - 晶体能量星

#### 星球特征
**环境设定**：
- **地貌类型**：晶体平原和能量裂谷为主
- **能量场**：遍布星球的自然能量源
- **气候条件**：能量波动影响的多变环境
- **资源特性**：以能量晶体和稀有矿物为主

**技术实现**：`ErekirPlanetGenerator.java`

#### 文明特点
1. **能量驱动**：基于晶体能量的高科技文明
2. **效率至上**：追求能量利用效率的最大化
3. **精密工程**：复杂的能量管理和分配系统
4. **创新激进**：不断推进技术边界的探索精神

#### 视觉风格定义
- **色彩基调**：晶体蓝、能量紫、金属银
- **材质特征**：水晶、合金、能量导体
- **建筑风格**：高科技感，流线型设计
- **环境氛围**：科幻感十足，能量流动的视觉效果

#### 代表性内容
**资源体系**：
- 基础晶体：铍、钨、氧化物
- 能量材料：碳化物、裂变物质
- 高级组件：休眠胞囊、晶体核心

**建筑特色**：
- 气流管道系统
- 热能传导网络
- 能量聚焦装置
- 高级防御系统

## 3. 背景故事与叙事设计

### 3.1 历史背景设定

#### 文明发展史
1. **古代时期**：各星球独立发展的本土文明
2. **接触时代**：星际航行技术的突破和首次接触
3. **扩张时期**：资源竞争驱动的星际殖民浪潮
4. **冲突时代**：当前游戏时间线，资源争夺的激化

#### 技术发展脉络
- **第一次技术革命**：基础工业和机械制造
- **第二次技术革命**：能源开发和电子技术
- **第三次技术革命**：自动化和人工智能
- **第四次技术革命**：星际航行和能量操控

### 3.2 冲突根源与动机

#### 资源稀缺性
**策划设定**：
- **基础资源枯竭**：传统工业资源的日益稀缺
- **新能源争夺**：对高效能源的迫切需求
- **技术垄断**：关键技术的控制权之争
- **生存空间**：适宜居住星球的有限性

#### 意识形态分歧
1. **工业主义 vs 能量主义**：不同的技术发展路线
2. **保守主义 vs 激进主义**：对技术应用的不同态度
3. **本土主义 vs 扩张主义**：对星际发展的不同策略
4. **集权主义 vs 分散主义**：不同的社会组织模式

### 3.3 叙事表达手法

#### 环境叙事
**技术实现**：地图设计、建筑布局、环境细节

1. **废墟遗迹**：通过损坏的建筑暗示过往冲突
2. **资源分布**：稀缺资源的位置反映争夺历史
3. **防御工事**：现有防御设施显示威胁性质
4. **技术遗留**：不同技术风格的建筑混杂

#### 隐含叙事
1. **任务描述**：简洁的文字暗示背景故事
2. **科技树**：技术发展路线反映文明选择
3. **单位设计**：不同单位体现的战术思想
4. **音效配乐**：营造特定的情绪氛围

## 4. 美术设计体系

### 4.1 整体美术风格

#### 设计定位
Mindustry采用**像素艺术**与**现代工业设计**相结合的视觉风格，在复古像素艺术的框架内表达现代工业美学。

#### 核心美术原则
1. **功能性优先**：视觉设计服务于功能识别
2. **信息层次清晰**：重要信息突出，次要信息隐退
3. **风格一致性**：所有元素遵循统一的设计语言
4. **可读性保证**：在任何缩放级别都保持清晰可读

### 4.2 色彩系统设计

#### 功能性色彩规范
**技术实现**：`Pal.java`、`Color.java`

1. **团队识别色彩**：
   - 蓝色系：玩家团队（友方）
   - 红色系：敌对团队（敌方）
   - 绿色系：中立团队（废弃）
   - 紫色系：特殊团队（Boss）

2. **资源类型色彩**：
   - 铜色：基础金属资源
   - 蓝色：液体和能量资源
   - 绿色：有机和生物资源
   - 紫色：稀有和高级资源

3. **状态指示色彩**：
   - 绿色：正常工作状态
   - 黄色：警告或低效状态
   - 红色：错误或危险状态
   - 灰色：停机或无效状态

#### 美学色彩搭配
1. **Serpulo配色方案**：
   - 主色：深灰蓝（#3c3c3c）
   - 辅色：工业黄（#f4d03f）
   - 强调色：橙红（#ff6b6b）

2. **Erekir配色方案**：
   - 主色：深紫蓝（#2d1b69）
   - 辅色：晶体蓝（#4ecdc4）
   - 强调色：能量紫（#a569bd）

### 4.3 图标设计体系

#### 设计原则
**技术实现**：`Icon.java`、`Iconc.java`

1. **简洁性**：使用最少的视觉元素传达概念
2. **一致性**：所有图标遵循相同的设计规范
3. **可识别性**：即使在小尺寸下也能清晰识别
4. **隐喻准确性**：图标与其代表的概念直接关联

#### 图标分类体系
1. **功能图标**：
   - 建造、移动、攻击、防御
   - 设置、信息、警告、错误

2. **资源图标**：
   - 物品类型：矿物、合金、液体、能量
   - 状态指示：满载、空置、流动、停滞

3. **建筑图标**：
   - 生产、传输、防御、支援
   - 工厂、炮塔、墙体、特殊设施

### 4.4 建筑视觉设计

#### 设计语言统一
1. **Serpulo建筑特征**：
   - 材质：金属板材、混凝土结构
   - 色彩：工业灰、铜绿、钢蓝
   - 形状：直角为主，功能性造型
   - 细节：螺栓、焊缝、工业管线

2. **Erekir建筑特征**：
   - 材质：晶体表面、能量导体
   - 色彩：深紫、晶蓝、银白
   - 形状：流线型、几何抽象
   - 细节：能量回路、晶体节点

#### 功能识别设计
1. **生产建筑**：
   - 视觉特征：排烟口、传送带接口
   - 动画效果：工作时的运转动画
   - 指示元素：生产进度、效率状态

2. **防御建筑**：
   - 视觉特征：装甲外观、武器轮廓
   - 动画效果：瞄准、射击、装填
   - 指示元素：射程显示、目标锁定

3. **传输建筑**：
   - 视觉特征：明显的方向性设计
   - 动画效果：物品移动、液体流动
   - 指示元素：流向箭头、堵塞警告

## 5. 单位视觉设计

### 5.1 设计分类体系

#### 按移动方式分类
1. **地面单位**：
   - 设计特征：履带、轮子、机械腿
   - 材质表现：重型装甲、机械结构
   - 动画特点：履带滚动、机械步态

2. **空中单位**：
   - 设计特征：螺旋桨、喷射口、翼面
   - 材质表现：轻量化外壳、推进系统
   - 动画特点：飞行姿态、引擎光效

3. **海军单位**：
   - 设计特征：船体结构、舰载武器
   - 材质表现：舰船装甲、防水涂装
   - 动画特点：航行波纹、炮塔转动

#### 按功能角色分类
1. **突击单位**：
   - 视觉特征：流线型外观、轻装甲
   - 配色方案：高对比度、警示色彩
   - 动作设计：快速移动、敏捷转向

2. **重装单位**：
   - 视觉特征：厚重装甲、多重武器
   - 配色方案：深色主调、威慑效果
   - 动作设计：稳重前进、威力射击

3. **支援单位**：
   - 视觉特征：功能性外观、辅助设备
   - 配色方案：中性色调、友好感受
   - 动作设计：协调配合、精确操作

### 5.2 进化设计理念

#### 技术层级视觉化
1. **T1单位**：简单几何形状，基础功能
2. **T2单位**：增加细节，功能专业化
3. **T3单位**：复杂结构，多功能集成
4. **T4单位**：精密设计，高科技感
5. **T5单位**：未来感造型，极致性能

#### 派系风格差异
1. **Serpulo派系**：
   - 造型风格：工业实用主义
   - 材质表现：钢铁、合金、实体装甲
   - 配色特点：军绿、钢灰、战术蓝

2. **Erekir派系**：
   - 造型风格：未来科技感
   - 材质表现：晶体、能量场、光效
   - 配色特点：深紫、电蓝、银光

## 6. 环境与特效设计

### 6.1 环境氛围营造

#### 星球环境差异化
**技术实现**：`Environment.java`、`Weather.java`

1. **Serpulo环境**：
   - 天空：工业雾霾、金属质感云层
   - 地面：岩石、金属地板、工业废料
   - 植被：稀少的改造植物、人工绿化
   - 音效：机械运转、工业噪音

2. **Erekir环境**：
   - 天空：能量极光、晶体反射光
   - 地面：水晶地貌、能量裂缝
   - 植被：晶体植物、发光生物
   - 音效：能量共鸣、晶体振动

#### 天气系统设计
1. **沙尘暴**：降低视野，影响精确度
2. **电磁干扰**：影响电子设备效率
3. **腐蚀雨**：对建筑造成缓慢损害
4. **能量潮汐**：周期性的能量增强

### 6.2 特效系统设计

#### 战斗特效
**技术实现**：`Fx.java`、`Effect.java`

1. **武器特效**：
   - 射击火花：不同武器的独特射击效果
   - 爆炸效果：分层爆炸、冲击波、碎片
   - 激光效果：能量束、充能过程、散射

2. **命中反馈**：
   - 装甲火花：显示伤害位置和强度
   - 护盾效果：能量护盾的波纹和闪烁
   - 结构损坏：建筑破坏的视觉表现

#### 环境特效
1. **工业特效**：
   - 生产烟雾：工厂运转的视觉指示
   - 传送带动画：物品移动的流畅表现
   - 能量流动：管道中资源的移动轨迹

2. **自然特效**：
   - 环境粒子：大气感的背景粒子
   - 光照变化：昼夜循环、动态光源
   - 天气效果：降雨、风暴、极光

## 7. UI界面设计

### 7.1 界面设计哲学

#### 功能优先原则
1. **信息层次**：重要信息优先显示
2. **操作效率**：减少不必要的操作步骤
3. **视觉负担**：避免过度装饰影响功能
4. **一致性**：所有界面遵循统一规范

#### 可用性设计
1. **直观性**：界面元素含义一目了然
2. **反馈性**：操作结果及时明确反馈
3. **容错性**：减少误操作的可能性
4. **学习性**：新用户能快速掌握操作

### 7.2 核心界面设计

#### 游戏主界面（HUD）
**技术实现**：`HudFragment.java`

1. **资源显示区**：
   - 位置：屏幕上方居中
   - 内容：当前拥有的各类资源数量
   - 设计：图标+数字，颜色编码

2. **建造菜单**：
   - 位置：屏幕右侧
   - 内容：可建造的建筑分类
   - 设计：分页标签，图标网格

3. **波次信息**：
   - 位置：屏幕左上角
   - 内容：当前波次、倒计时、敌人信息
   - 设计：紧急感的红色基调

#### 建造界面设计
1. **分类导航**：
   - 按功能分类：生产、防御、传输、特殊
   - 图标设计：直观的功能图标
   - 颜色编码：快速识别不同类别

2. **建筑信息**：
   - 即时预览：建造范围、连接关系
   - 成本显示：所需资源清单
   - 状态提示：是否可建造、缺少什么
